在这个工作区里，Publish-All、公众号工具链和小红书 skill 都把它视为长期宿主环境。
推荐安装流程
先保稳定，再追求自动化
01 先用 nvm 准备 Node LTS 与 npm 02 npm i -g openclaw 03 先尝试 non-interactive，失败时回退到交互 OAuth 04 安装后立刻 gateway install + start 05 用 status、dashboard 与 token 验证连通
nvm use --lts · openclaw onboard · openclaw gateway start
第一次上手最容易踩的坑
brew 需要 sudo
没有管理员权限的机器上，优先改走 nvm + npm 路线。
OAuth 不能强行无交互
新版本下遇到失败时，要允许回退到浏览器交互授权。
gateway 未启动
只做 onboarding 不启动 gateway 时，status 容易显示 unreachable。
排错顺序建议
先看 nvm / node 是否可用，再确认 OAuth 是否需要交互回退，然后检查 gateway，最后处理 devices approve / clear / repair。
在这个工作区里可以怎么用
它既是技能宿主，也是连接命令行与 MCP 的中枢
First Commands To Remember
openclaw status 查看当前状态 openclaw dashboard 打开本地 dashboard openclaw devices approve --latest 处理修复或升级后的配对请求 openclaw tui --token ... 以终端方式连接与操作
官方截图也说明了它并不只是安装工具，而是一个真正承接多界面、多通道和自动化工作流的长期运行环境。
